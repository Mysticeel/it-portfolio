Certification reflection

What passing AZ-900 with 879 taught me

AZ-900 gave me a clear model of how Azure services fit together, but the biggest lesson was that knowing the name of a service is not the same as being able to operate it.

My next step is therefore practical: build the services, break the configuration, troubleshoot the result and document what I learn. The certification is the starting point for the work, not the finish line.

Lab design

Designing an Azure home lab that proves real skills

The lab will build in layers: resource groups and tags, storage, virtual networks, NSGs, virtual machines, identity, monitoring, backup and finally Bicep.

Every lab must answer three questions: what business problem does this solve, how did I validate it, and what would I change in production?

The output will be more than screenshots. Each project will include a README, an architecture summary, deployment steps, validation evidence, lessons learned and a clean-up plan to control cost.

Project journal

Building CloudOps Toolkit with a professional workflow

CloudOps Toolkit is deliberately being built with feature branches, pull requests, milestones, architecture decisions and a product backlog. That process is part of the portfolio evidence.

The project currently includes an application shell, interactive menu, settings management, logging and an About screen. The next modules will focus on useful cloud administration and reporting tasks.

Automation case note

Turning new starter administration into a repeatable workflow

A typical onboarding task crosses multiple systems: Active Directory, group membership, directory synchronisation and Microsoft 365. The automation approach separates user input, role configuration, validation and cloud-side actions.

The important engineering lesson is that waiting, retries and clear failure messages matter as much as the happy path. Cloud synchronisation is not immediate, so the workflow must expect delay rather than treat it as an error.
