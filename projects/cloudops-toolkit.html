Featured project case study

CloudOps Toolkit

A practical Python administration toolbox built to demonstrate cloud operations, Microsoft 365, identity, automation, reporting and professional software delivery.

View repository ↗Back to projects

The problem

Cloud and Microsoft 365 administrators repeatedly gather inventory, inspect users, review licences, run scripts and produce reports. These tasks are often spread across portals and one-off commands.

CloudOps Toolkit explores how those actions could be brought into a consistent, documented interface while providing a realistic project through which to practise Python, PowerShell, APIs and DevOps habits.

What exists today

  • Structured repository and source layout.
  • Application shell and interactive menu.
  • Central configuration through environment settings.
  • Basic application logging.
  • About screen and project documentation.
  • Roadmap, changelog, contributing guide, ADRs and product backlog.

Planned capability

Azure

Resource inventory, VM administration, cost visibility and reporting.

Microsoft 365

User and licence reporting through Microsoft Graph.

Entra ID

User, group and identity administration modules.

Engineering approach

The repository uses feature branches, pull requests, milestones and clean commits. Documentation is treated as part of the product rather than something added at the end.

The portfolio value is not only the tool. It is the evidence of planning, version control, debugging, documentation and incremental delivery.
